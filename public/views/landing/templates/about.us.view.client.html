This WebApp is submitted as part of the project for CS 5610 Web Development.

The Tech stack used for the Project is:
  1. DATABASE: MongoDB
  2. NodeJS as Server
  3. ExpressJS as Framework for building API
  4. Bootstrap for building a rich UI interface
  5. AngularJS for seameless routing and state persistence in the frontend
  6. MongooseJS as the ODM.
Internal Details and Citation:
  1. Angular UI-Router for routing
  2. BodyParser for parsing the request body/headers
  3. The Theme used for building the website is 'FLATLY'. Link: Flatly
  4. OAuth 2.0 implemented using OAuth2orize

How to use (For a Normal User)

  1. Register in OAuthorizer
  2. Update your profile to reflect your latest information
  3. Visit the web application/service which you want to use
  4. Go to the Login page of the application
  5. While logging in click on the button which says "Login with Sanjay's Website"
  6. You will be redirected to the login page of OAuthorize
  7. Login into OAuthorize
  8. You will be asked to "Allow" or "Deny" permission for the application to use your profile data
  9. If you "Deny" , you will be redirected to the OAuthorize login page.
  10. If you choose to "Accept" , means you have authorized the application/service to access your profile data on your behalf
  11. You are taken to the application/service, where you would be able to see your profile data.

How to use (For an Application Owner)

  1. Register in OAuthorizer as an "App Owner"
  2. Update your profile to reflect your latest information
  3. Make sure your "Application Specific" settings are upto date.
  4. The Client ID, Secret and RedirectURI are very important and they must reflect the latest information
  5. Place a button which says "Login with Sanjay's website" in your login page
  6. Implement a service on the server side, which redirects to OAuthorize with your ClientID and RedirectURI
    Note: These must be exactly similar to the ones you have configured here!
  7. Implement a Callback service which takes care of the RedirectURI, Use the Authorization Code you received from OAuthorizer as a Bearer token to get your access token from OAuthorizer
  8. Now that you have got the Access token, you can get the user's profile data from 'api/oauth/user/< userid >' by making a request to OAuthorizer with your Client ID and secret as username and password for a Basic Authentication by passing them in the Authentication header.
  9. You will receive all the user's information that the user was ready to share with you. Use wisely. With great power comes great responsibility!
  10. Enjoy hassel free authentication!
